App Structure & Wireflows: Adaptable NYC Focus

Exploring architectural patterns to incorporate rich, location-specific data (NYC currents map, time warp, local commentary). Core Philosophy: Highly specific NYC knowledge (e.g., "flooding flows west to east") and Time Warp UI logic are hard-coded in React components targeted at /app/nyc. Generic locations fallback to a simpler view without complex routing or over-engineered API feature flags.

1. Hub and Spoke (The "Dive Deeper" Model)

The main location page acts as a quick operational dashboard. For NYC, a hard-coded "Dive Deeper" button routes the user to a dedicated /app/nyc/currents React component containing the Time Warp tool, legacy maps, and specific Grimaldo's commentary. For a generic location, the button simply isn't rendered.

2. Segmented Tabs (The "Context Switcher")

Everything lives on /app/{loc}. For NYC, a dedicated "Water Data" tab holds the hardcoded time-warp and complex plots alongside the live condition. For locations lacking this data, the tabs either hide entirely or the "Water Data" tab only shows simple tide lines.

4. Responsive Overlay (The "App-Native" Model)

Inspired by Claude's explorations. The main location page remains a clean dashboard. Deeper content like the Time Warp and NYC's local maps are housed in a responsive overlay. On mobile, this slides up as a full-screen sheet. On desktop, it slides in as a side drawer. This keeps the user on a single URL without navigating away or causing the main layout to jump around.
